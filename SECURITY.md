# Politique de Securite

## Versions Supportees

| Version | Supportee |
| ------- | --------- |
| 2.x     | Oui       |
| 1.x     | Non       |

## Signaler une Vulnerabilite

Si vous decouvrez une vulnerabilite de securite dans projinit, merci de la signaler de maniere responsable.

### Processus

1. **Ne pas** ouvrir une issue publique
2. Envoyez un email a `xgueret@users.noreply.github.com` avec:
   - Description de la vulnerabilite
   - Etapes pour reproduire
   - Impact potentiel
   - Suggestion de correction (si possible)
3. Attendez une reponse sous 48h
4. Coordonnez la divulgation avec les mainteneurs

### Ce que nous promettons

- Reponse initiale sous 48 heures
- Mise a jour reguliere sur l'avancement
- Credit dans les notes de version (si souhaite)
- Pas de poursuites pour les signalements de bonne foi

### Hors perimetre

Les problemes suivants ne sont generalement pas consideres comme des vulnerabilites :

- Bugs sans impact securitaire
- Problemes de performance
- Suggestions d'amelioration

## Bonnes Pratiques

Lors de l'utilisation de projinit :

- Ne commitez jamais de secrets dans votre configuration
- Utilisez `.gitignore` pour exclure les fichiers sensibles
- Verifiez les templates avant de les utiliser en production
- Maintenez projinit a jour

## Historique

Aucune vulnerabilite signalee a ce jour.
